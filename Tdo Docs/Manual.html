Выражения


Типы выражений


Выражения используются в TDO во многих случаях: вычисляемые поля, выражения
фильтра, списки полей сортировки, валидаторы.

В TDO выражения подразделяются на арифметичечкие,строковые, сравнения, логические и списковые.

Арифметическое выражение — это выражения, результатом вычисления которого
является число.

Строковое выражение — это выражения, результатом вычисления которого
является строка знаков.

Выражение сравнения — это выражение, результатом вычисления которого
является булевое значение true или false.

Логическое выражение — это выражение, результатом вычисления которого
является булевое значение true или false.

Использование круглых скобок

В выражении могут использоваться круглые скобки для группирования частей выражения и/или изменения порядка операций с любым уровнем вложенности, например:



((A+6) * 4 > B) AND (C+4) = 7



Арифметические выражения

Результатом арифметического выражения может быть целое значение типа java.util.Long, число с плавающей запятой типа java.util.Double или значение типа java.math.BigDecimal.

Операторы


Допустимыми арифметическими операторами выражения являются:

  • Сложение (+), например: column + 7
  • Вычитание (-), например: column — 7
  • Умножение (*), например: column * 7
  • Деление (/), например: column / 7

Операнды

Операндом арифметического выражения может быть числовой литерал, идентификатор или арифметическое выражение.

Числовой литерал может иметь тип int,long, float или double, например:

5 + 6L — 7.1F * 9D

Идентификатор в TDO framework может быть именем колонки схемы таблицы или параметром. Параметр — это идентификатор с преффиксом равным символу : (двоеточие). Имя колонки или параметр должны определять данные типа java.util.Byte, java.util.Short, java.util.Integer, java.util.Long, java.util.Float, java.util.Double, java.math.BigDecimal.

Использование скобок в выражении



В выражении могут использоваться круглые скобки для выделения и группирования частей выражения с любым уровнем вложенности, например:

(( a + b ) * c ) /100



Преобразование типов



В процессе вычисления значения арифметического выражения могут производится преобразования типов операндов выражения. Неформально правила преобразования могут быть представлены следующим образом:

  • Если оба операнда в простом выражении имеют целочисленный тип, то они преобразуются к типу Long и результат операции будет Long. Исключением является оператор деления. В таком случае оба операнда преобразуются к типу Double и результатом является тип Double.

  • Если один из операндов имеет тип BigDecimal, то второй преобразуется к типу BigDecimal и результат будет BigDecimal.

  • Если один из операндов имеет тип с плавающей точкой, то оба операнда преобразуются к типу Double и результатом будет тип Double.

Допустимы также данные типа String, содержимое которых представляет собой правильное число.

Примеры.

'273' + 6

Результат 279

'273' + '6 '

Результат 279

Особый случай, когда значение одного из операндов или обоих сразу равно null. В этом случае операнды с null значениями считаются, как имеющие тип Integer со значением равным Integer(0) и дальнейшие действия по преобразованию типов проводятся согласно правилам, перечисленным выше.

Строковые операторы


Существует единственный строковый оператор, используемый в TDO.

  • Конкатенация (|| ), например: 'ab' || 'bc' в результате дает строку 'abcd'.

Синтаксис:



expr-1 || expr-2

Если один или оба операнда равны null, то перед выполнением операции они преобразуются в пустую строку знаков, т.е. объект типа java.lang.String нулевой длины.


При необходимости операнд(ы) преобразуюся в строку(и).


Пример.


'abc' || '12'

'abc' || 12


оба выражения дают одинаковый результат: 'abc12'


Операторы сравнения


Допустимыми операторами сравнения в выражении являются:

  • Больше ( > )
  • Меньше ( < )
  • Равно ( = )
  • Не равно ( <> )
  • Больше или равно ( >= )
  • Меньше или равно( <= )
  • Проверка на равенство значению null ( is null) , например: Col2 is null
  • Проверка на не равенство значению null ( is not null ), например: Col2 is null
  • Проверка на принадлежность значения заданному интервалу ( between ) например: Col2 between 36 and 58
  • Проверка на равенство значения одному из элементов списка ( in ) например: Col2 in ( 1,2,36,58)
  • Проверка на равенство начальной подстроки строкового значения заданной строке ( starting with ), например: Col2 starting with 'Mr.'
  • Проверка, содержит ли строка заданную подстроку (containing), например: Col2 containing 'Mr.'
  • Проверка на соответствие wildcard-шаблону ( like ), например: Col2 like '%Mr%'
  • Проверка на соответствие регулярному выражению ( regex ), например: Col2 regex '[0-9]{1,6}'

Примечание. Операторы могут быть набраны в любом регистре, например:
Starting With или StArTiNg WiTh

Преобразование типов



В процессе вычисления значения операции сравнения могут производится преобразования типов операндов выражения.

Для стандартных операций сравнения, таких как <,<=,>,>=,=,<> действуют следующие правила:

Примечание. Для ясности и общности будем считать, что при сравнении результат получается одним из 0 или 1 или -1, как для методов compareTo.

  • Если оба операнда равны null, то результат всегда 0;

  • Если первый операнд равен null, а второй отличен от null, то результат всегда -1;

  • Если второй операнд равен null, а первый отличен от null, то результат всегда 1;

  • Если первый операнд равен второму в смысле java-операции сравнения == , то считается, что операнды равны ( результат 0) ;

  • Если один из операндов имеет тип java.math.BigDecimal, то другой операнд преобразуется, если необходимо, в тип java.math.BigDecimal ;

  • Если один из операндов имеет тип java.lang.Double, то другой операнд преобразуется, если необходимо, в тип java.lang.Double;

  • Если один из операндов имеет тип java.lang.Float, то оба операнда преобразуются,в тип java.lang.Double;

  • Если тип одного из операндов java.lang.Byte, java.lang.Short, java.lang.Integer, то оба операнда преобразуются, если необходимо, в тип java.lang.Long;

  • Если оба операнда имеют тип java.util.Date, то результат определяется методом Date.compareTo(Date);

  • Если оба операнда имеют тип java.sql.Timestamp, то результат определяется методом Timestamp.compareTo(Timestamp);

  • Если первый операнд имеют тип java.sql.Timestamp, а второй операнд java.util.Date, то результат аналогичен выполнению метода ((Timestamp)firstValue).compareTo((Date)secondValue).

  • Если первый операнд имеют тип java.util.Date, а второй операнд java.sql.Timestamp, то результат аналогичен выполнению ((Timestamp)secondValue).compareTo((Date)firstValue) и, если значение результата равно 0, то возвращается 0. Иначе возвращается результат с противоположным знаком;

  • Если один из операндов имеет тип java.util.Date, а второй операнд java.lang.String, то строковый операнд преобразуется в тип java.util.Date. Результат аналогичен выполнению метода Date.compareTo(Date);

  • Если один из операндов имеет тип java.sql.Timestamp, а второй операнд java.lang.String, то строковый операнд преобразуется в тип java.sql.Timestamp и Результат аналогичен выполнению метода java.sql.Timestamp.compareTo(java.sql.Timestamp);

  • Если оба операнда имеют тип java.lang.String то возвращается результат выполнения метода String.compareTo(String).

  • Если первый операнд имеет тип java.lang.String, то второй преобразуется в строку методом toString() и возвращается результат выполнения метода String.compareTo(String);

  • Если второй операнд имеет тип java.lang.String то первый преобразуется в строку методом toString() и возвращается результат выполнения метода String.compareTo(String).

  • Если выполнено условие firstValue.getClass().equals(secondValue.getClass()) ); и если firstValue.equals(secondValue), то возвращается 0. В противном случае, т.е. ни одно из условий выше не выполнено, возвращается -1.



Операция BETWEEN

Синтаксис:

expr [NOT] BETWEEN expr-1 AND expr-2



Проверяет, лежит ли значение выражения expr в интервале ограниченном левой границей expr-1 и правой expr-2 , включая сами границы.



В выражении BETWEEN участвуют три операнда. В процессе вычисления могут производится преобразования типов. Первый операнд сравнивается со вторым по правилам ">=" - больше или равно, а также с третьим по правилам "<=" - меньше или равно. Преобразование типов, при этом, проводится как для стандартных операций сравнения.

Пример 1.

'12' BETWEEN 11 AND '20'

12 BETWEEN 11 AND 20

Результат равен true.

Пример 2.

'12' BETWEEN 11 AND '2'

Результат равен true. Это потому, что, как мы знаем, строка '12' сравнивается со строкой '2' по правилам для стандартных операций сравнения. Но тогда сравнение производится методом java.lang.String.compareTo. В этом случае, разумеется, строка '2' больше строки '12'.

Операция NOT BETWEEN возвращает true, когда операция BETWEEN возвращает false и наоборот.

Операция IN

Синтаксис:



expr [NOT] IN ( expr-1, expr-2, ..., expr-n )

Проверяет, содержится ли значение выражения expr в списке. Элементы списка expr-1, ..., expr-n также могут быть выражениями.

Если expr равен null, то, если список содержит выражение со значением null, результат будет true.

Тип выражения expr может отличаться от типа элементов, а каждый элемент может быть любого типа. Сравнение производится по-элементно на равенство. При этом, для каждого сравнения может производится преобразование типов, как принято для стандартной операции = 'равно'.

Пример.

'12' IN ( 3, '7' ,12)

Результат равен true, хотя в списке содержится целое значение 12, проверяемое строковое значение преобразуется в Integer.

Операция NOT IN возвращает true, когда операция IN возвращает false и наоборот.



Операция Starting With

Синтаксис:

str-expr [NOT] Starting With str-expr-1



Проверяет, содержится ли в начале значения выражения expr в подстрока str-expr-1.

Если expr или str-expr-1 или оба равен/равны null то, выражение равно false.

Перед выполнением сравнения оба операнда выражения преобразуются, если необходимо, в строку знаков.

Пример.

'1234' starting with 12

Результат равен true. Целое значение 12 второго операнда преобразуется в String '12'.

Пример.


null starting with 12

null starting with null

'12' starting with null

Результат всегда равен false.



Операция NOT STARTING WITH возвращает true, когда операция STARTING WITH возвращает false и наоборот.



Операция Containing

Синтаксис:

str-expr [NOT] CONTAINING str-expr-1



Проверяет, содержится ли в строке expr подстрока str-expr-1.

Если expr или str-expr-1 или оба равен/равны null то, выражение равно false.

Перед выполнением сравнения оба операнда выражения преобразуются, если необходимо, в строку знаков.

Пример.

'1234' containing 23

'1234' containing 34

'1234' containing '1234'

Результат равен true.

Пример.


null containing 12

null containing null

'12' containing null

Результат всегда равен false.



Операция NOT CONTAINING возвращает true, когда операция CONTAINING возвращает false и наоборот.



Операция LIKE

Синтаксис:

str-expr [NOT] LIKE str-expr-1

Выполняет сравнение str-expr по заданному образцу str-expr-1.

str-expr-1 это строковое выражение, содержащее образец.



Условия LIKE предназначены для сопоставления символьных строк с заданным образцом.

Символы, представляющие значение str-expr-1 трактуются:

  • Символ подчеркивания "_" обозначает любой отдельный символ;

  • Знак процента "%" представляет любую последовательность из k символов (где k может быть равно нулю);

  • Все другие символы обозначают сами себя.

Пример 1. Выражение:

'Bill Gates' LIKE '%Gates'

Даст true для приведенного выше выражения. Строка str-expr слева от ключевого слова LIKE может содержать любые символы перед словом 'Gates'.

Пример 2. Выражение:

'Bill Gates' LIKE 'B%ates'

Даст true для приведенного выше выражения. Строка str-expr слева от ключевого слова LIKE может содержать любые символы между символом 'B' и строкой 'ates', например, 'Bob Lates' LIKE 'B%ates' также даст true.

Пример 3. Выражение:

'Bill Gates' LIKE 'Bill _ates'

Даст true для приведенного выше выражения. Строка слева от ключевого слова LIKE может содержать любой символ вместо символа 'G', например, 'Bill Lates' LIKE 'Bill _ates' также даст true.

Пример 4. Выражение:

'Bob Gates' LIKE 'B% Gate_'

Даст true для приведенного выше выражения. Строка слева от ключевого слова LIKE может содержать любую строку после символа 'B', затем следует пробел, строка 'Gate' и любой символ.

Операция NOT LIKE возвращает true, когда операция LIKE возвращает false и наоборот.



Операция RegEx

Синтаксис:

str-expr [NOT] REGEX str-expr-1

Выполняет сравнение str-expr по заданному образцу str-expr-1.

str-expr-1 это строковое выражение, содержащее регулярное выражение.



Пример 1. Выражение:

'B. Gates' regex '(B(ill)?|B.) Gates'

Вернет true для приведенного выше выражения. Точно также вернет true выражение:

'Bill Gates' regex '(B(ill)?|B.) Gates'

Пример 2. Выражение:

'1' RegEx '[0-9]{1,5}'

'12' RegEx '[0-9]{1,5}'

'123' RegEx '[0-9]{1,5}'

'1234' RegEx '[0-9]{1,5}'

'12345' RegEx '[0-9]{1,5}'

Все приведенные выражения дают при вычислении true. В них проверяется левый операнд на то, что его символы должны быть цифрами и количество цифр от 1 до 5. Так что следующее выражение даст false.

'123456' RegEx '[0-9]{1,5}'



Операция NOT REGEX возвращает true, когда операция REGEX возвращает false и наоборот.

выражение, значением которого является объект типа tdo.expr.ValueList.

Функции в выражениях

Выражение может содержать в качестве операндов вызовы функций. Функция состоит из идентификатора и списка параметров, разделенных запятыми. Список параметров заключен в круглые скобки. В качестве параметров могут использоваться ваыражения. Состав параметров, их количество и типы определяются синтаксисом и семантиков конкретной функции.

Условно функции можно разделить на группы:

  • Функции обработки строк

  • Функции преобразования типов

  • Функции состояния

  • Специальные функции

  • Функции пакета java.Math

Функция UpperCase

Синтаксис:

UpperCase( str-expr )

Алиасные имена: U, Upper

Переводит символы строки в верхний регистр.

Если значение str-expr равно null, то преобразования не производится. Для остальных значений выражение преобразуется в строку применением метода toString().

Функция LowerCase

Синтаксис:

LowerCase( str-expr )

Алиасные имена: L, Lower

Переводит символы строки в нижний регистр.

Если значение str-expr равно null, то преобразования не производится. Для остальных значений выражение преобразуется в строку применением метода toString().



Функция Proper

Синтаксис:

Proper( str-expr )

Алиасные имена: P

Переводит самый первый символ строки в верхний регистр.


Если значение str-expr равно null, то преобразования не производится. Для остальных значений выражение преобразуется в строку применением метода toString().



Функция Header

Синтаксис:

Header( str-expr [ ,delimeters] )

Алиасные имена: Hd

Переводит самый первый символ каждого слова строки в верхний регистр.


Если значение str-expr равно null, то преобразования не производится. Для остальных значений выражение преобразуется в строку применением метода toString().

Параметр delimeters определяет символы-разделители слов в строке. Если delimeters опущен, то по умолчанию он принимает строковое значение: " .,:-_;", или пробел,точка,запятая,двоеточие,дефис,подчеркивание, точка с запятой.



Функция Trim

Синтаксис:

Trim( str-expr)

Алиасные имена: TR

Отсекает крайние символы пробела слева и справа строки.

Если значение expr равно null, то преобразования не производится. Для остальных значений выражение преобразуется в строку применением метода toString().

Функция NumberFormat

Синтаксис:

NumberFormat(expr , [ pattern [, decseparator[ , groupseparator ]] ] )


Алиасные имена: NumFormat, NF

Форматирует числовой операнд expr согласно заданному шаблону pattern и преобразует его в строковое значение.

По умолчанию pattern равен "###.########", разделитель целой и дробной частей decseparator равен '.' - точка, и разделитель групп groupseparator цифр равен ' ' - пробелу.


Если значение expr равно null, то преобразуется в Integer со значение 0.


Функция DateFormat

Синтаксис:

DateFormat( date-expr [ [ , pattern [ , localeLang ] ] )

Алиасные имена: DF

Преобразует заданное значение date-expr в тип java.lang.String по заданному образцу pattern и заданому localeLang.


Параметры:

  • date-expr — выражение типа java.util.Date, которое преобразуется в java.lang.String.

  • pattern - строковое выражения, представляющее собой шаблон (образец) согласно которому дата должна быть представлена как строка.

  • localeLang - строковый код страны, как определено конструктором класса java.util.Locale. Например «ru» - русский, а «en» - английский.



Если pattern не задан, то преобразование производится с использованием значения равного "dd.MM.yy".

Если не задан localeLang, то преобразование выполняется согласно текущим языковым установкам операционной системы компьютера. Для правильного использования функции необходимо точно знать какой язык использован для представления даты в формате строки знаков.


Значение date-expr равное null, преобразуется в java.util.Date со значением Date(0) перед преобразованием в строку знаков.

Преобразование проводится с помощью класса java.text.SimpleDateFormat.




Пример. Пусть строка date-expr содержит значение типа java.util.Date, соответствующее 12 июля 2005.


Для такой даты применим шаблон 'MMM, d yyyy'.


Если выполнить выражение DateFormat( date-expr, 'MMM, d yyyy') , то для Windows с русскими языковыми установками возвращается «Июл, 12 2005».


Если выполнить выражение DateFormat(date-expr, 'MMM, d yyyy','en') , то возвращактся строка «Jul, 12 2005», поскольку третьим параметром мы явно проинформировали функцию, на каком языке представлена дата — английском.
